# PrismStar — v2 要件定義（動線・画面・データモデル）

PrismStar v1（ログイン・作品投稿・GitHub 連携・スター・公開フィード）の上に、
**マイページ（管理）／作品の公開・非公開／作品の作成・編集／タグ／検索** を載せる v2 の要件定義。

- 現状の技術仕様（v1 実装済み）：`docs/spec.md`
- 設計判断と理由：`docs/decisions.md`（ADR-013〜017）
- 実装の段取り：`docs/roadmap.md`（Phase 6〜11）

---

## 1. 用語（先に固定）
- **マイページ（＝管理画面）**：本人だけの管理ページ。自分の**全作品（非公開含む）**を一覧し、新規作成・編集・削除・公開/非公開切替・GitHub 設定を行う。
- **ユーザーページ（＝公開プロフィール）**：他人が見るページ。そのユーザーの**公開作品のみ**表示。
- **作品の状態**：**公開 / 非公開** の2状態（[ADR-014](decisions.md)）。非公開＝本人のみ閲覧（＝下書き兼用）。

## 2. 動線（サイトマップ）
```
未ログイン:
  ホーム/一覧 ── 検索 ── ログイン ── 新規登録
       └→ 作品詳細(公開のみ) ── ユーザーページ(公開作品のみ)

ログイン後:
  ホーム/一覧 ── 検索 ── マイページ(管理) ── ログアウト
                              │
              ┌───────────────┼────────────────┐
        新規作成→[作成画面]   全作品一覧→[編集画面]    GitHub設定
            保存 / 戻る        保存 / 削除 / 戻る / 公開切替
```

## 3. 画面と操作
| 画面 | ファイル(予定) | 認証 | 操作・表示 |
|---|---|---|---|
| 新規登録 | `register.php` | 未 | 登録 → 自動ログイン |
| ログイン | `login.php` | 未 | ログイン |
| ホーム / 一覧（公開フィード） | `gallery-list.php` | 任意 | 公開作品の新着グリッド |
| 検索 | `search.php`（新規） | 任意 | ワード/タグで公開作品（**自分は除く**）＋ユーザー検索 |
| 作品詳細 | `gallery-detail.php` | 任意 | 公開＝誰でも／非公開＝本人のみ。所有者には編集導線 |
| ユーザーページ（公開） | `profile.php` | 任意 | そのユーザーの**公開作品のみ**＋獲得スター |
| **マイページ（管理）** | `mypage.php`（新規） | 要 | 自分の**全作品**一覧＋新規作成・編集・削除・公開切替・GitHub設定 |
| 作成画面 | `work-edit.php`（新規・兼用） | 要 | 入力（タイトル/説明/画像/タグ）＋公開/非公開選択＋**保存**／**戻る** |
| 編集画面 | `work-edit.php?id=N` | 要 | 内容編集＋公開切替＋**保存**／**削除**／**戻る** |

作成・編集はモーダルから**専用フォーム画面**へ格上げ（画像アップロード＋公開設定＋タグが乗るため）。

## 4. データモデル差分
v1 スキーマ（`docker/init.sql`）に対する追加。冪等 migration（`src/migrate.php`）＋ `init.sql` 反映。

- **`gallery.visibility`** … `ENUM('public','private')` 既定 `'public'`（[ADR-014](decisions.md)）。
- **`tags`**（マスタ）… `id` PK / `name`(UNIQUE) / `created_at`（[ADR-017](decisions.md)）。
- **`gallery_tags`**（中間・多対多）… `gallery_id` FK / `tag_id` FK / `PRIMARY KEY(gallery_id, tag_id)`、両 FK `ON DELETE CASCADE`。
- **画像アップロード**（[ADR-015](decisions.md)）… `public/uploads/` に保存し `gallery.src` にパスを持つ（外部 URL も併存可）。スキーマ変更は不要（`src` を流用）。

## 5. 可視性ルール
| 作品状態 | ホーム/一覧 | 検索 | ユーザーページ | 作品詳細 | マイページ |
|---|---|---|---|---|---|
| 公開 | ○ | ○（自分は除く） | ○ | 誰でも | ○ |
| 非公開 | × | × | × | **本人のみ** | ○（本人） |

**非公開作品を所有者以外に返さないことを safety invariant とする**（一覧・検索・ユーザーページ・詳細すべてでガード）。

## 6. API 差分（予定）
| メソッド / エンドポイント | 機能 | 認証 |
|---|---|---|
| `GET /api/gallery.php` | 公開フィード（`visibility='public'`、author/star/tags 同梱） | 不要 |
| `GET /api/gallery.php?id=N` | 詳細（公開＝誰でも／非公開＝本人のみ、他は 404） | 任意 |
| `GET /api/gallery.php?mine=1` | 自分の全作品（非公開含む。マイページ用） | **要** |
| `POST /api/gallery.php` | 作成（title/description/visibility/tags/画像 or URL） | **要** |
| `PATCH /api/gallery.php?id=N` | 編集（所有者のみ。内容・公開切替・タグ・画像） | **要** |
| `DELETE /api/gallery.php?id=N` | 削除（所有者のみ） | **要** |
| `GET /api/users.php?id=N` | ユーザーページ（公開作品のみ＋総スター。email/pass は返さない） | 任意 |
| `GET /api/search.php?q=&tag=&type=works\|users` | 検索（works＝公開かつ自分以外、title/desc/tag／users＝name/github） | 任意 |
| `POST/DELETE /api/stars.php` | スター付与/解除（既存） | **要** |

## 7. 認証境界
- **閲覧（公開コンテンツ）**：未ログインでも可（ホーム・検索・ユーザーページ・公開作品詳細）。
- **書き込み・管理**：作成・編集・削除・公開切替・スター・マイページは要ログイン（`$_SESSION['user_id']`）。

## 8. セキュリティ（v2 で増える不変条件）
既存の safety invariants（`docs/ai-roles-and-workflow.md` §4）に加えて：
- **非公開作品は所有者以外に一切返さない**（全取得経路でフィルタ／詳細で所有者判定）。
- **編集・削除・公開切替は所有者のみ**（`WHERE id=? AND user_id=?`）。
- **アップロード検証**：MIME/拡張子（jpg/png/webp/gif 等）・サイズ上限・ファイル名はサーバ生成・保存先固定・実行可能ファイル拒否（パストラバーサル防止）。
- 既存（プリペアドステートメント・`password_hash`/`session_regenerate_id`・GitHub token サーバ側秘匿）は維持。

## 9. スコープ外（今回はやらない）
- **GitHub リポジトリの作品テーブル永続化**（[ADR-016](decisions.md)）。GitHub repo は表示レイヤーのまま（プロフィール/マイページで都度取得）。
- **限定公開（URL を知る人のみ）**（[ADR-014](decisions.md) で却下。要件が出れば再検討）。
- ページネーション・虹色テーマ等の作り込み（`docs/roadmap.md`「今後の発展」）。
